Go from zero to a production AI incident response assistant that answers **"what happened, why did it happen, and how do we fix it?"** in four phases. Start with a single incident report and a real search query. End with a FastAPI backend that ingests incidents, runbooks, logs, alerts, dashboards, and postmortems, then generates GPT-4o-powered investigations grounded in your organization's operational knowledge.

> **How this guide is structured.** Each phase ends with something that works. Phase 0 is a complete minimal system in under 10 minutes. Phases 1–3 progressively add richer operational context, automated investigation, and production-ready workflows. You never need to redo what came before.

> **All code in this cookbook is real.** Base URL is `https://api.hydradb.com`. Get your API key at [app.hydradb.com](https://app.hydradb.com).

## Phase 0 - Minimal Working System · 5–10 minutes

The only goal is to see a real investigation response from a real incident document you uploaded. No multi-source ingestion, no backend server, no automated triage, and no chat interface—just the four API calls that prove the incident investigation pipeline works end-to-end.

**Goal:** Upload a single incident document, run one investigation query, and retrieve grounded results.

> ⚠️ **Complete Phase 0 first, even if you plan to skip ahead.** Every later phase assumes your incident knowledge database exists and indexing is working correctly. Running Phase 0 takes less than 10 minutes and eliminates the most common setup issues before moving to automated incident analysis.

## Step 2 - Add Incident Metadata Parsing

Each incident file contains structured fields near the top:

```text
Service: Checkout API
Severity: SEV-2
Date: 2026-06-12
Status: Resolved
```

Create `phase1/ingest_incidents.py`:

```python
from __future__ import annotations

import re
from pathlib import Path
from typing import Any

from config import DATABASE_ID, client


PROJECT_ROOT = Path(__file__).resolve().parents[1]
INCIDENTS_DIR = PROJECT_ROOT / "data" / "incidents"

INCIDENT_ID_PATTERN = re.compile(r"^# Incident\s+([A-Za-z0-9_-]+)", re.MULTILINE)


def extract_field(content: str, field_name: str) -> str:
    """
    Extract a metadata field from an incident Markdown document.

    Example:
        Service: Checkout API
    """
    pattern = re.compile(
        rf"^{re.escape(field_name)}:\s*(.+)$",
        flags=re.MULTILINE | re.IGNORECASE,
    )

    match = pattern.search(content)
    return match.group(1).strip() if match else "unknown"


def parse_incident(path: Path) -> dict[str, Any]:
    """
    Read one incident report and convert it into a HydraDB ingestion item.
    """
    content = path.read_text(encoding="utf-8")

    incident_match = INCIDENT_ID_PATTERN.search(content)
    incident_id = incident_match.group(1) if incident_match else path.stem

    service = extract_field(content, "Service")
    severity = extract_field(content, "Severity")
    incident_date = extract_field(content, "Date")
    status = extract_field(content, "Status")

    return {
        # Stable IDs prevent the same file from creating uncontrolled duplicates.
        "id": f"incident/{incident_id}",
        "type": "incident_report",
        "content": {
            "text": content,
        },
        "metadata": {
            "incident_id": incident_id,
            "service": service,
            "severity": severity,
            "incident_date": incident_date,
            "status": status,
            "source_path": str(path.relative_to(PROJECT_ROOT)),
        },
    }


def load_incidents() -> list[dict[str, Any]]:
    """
    Load every Markdown incident report from data/incidents.
    """
    paths = sorted(INCIDENTS_DIR.glob("*.md"))

    if not paths:
        raise FileNotFoundError(
            f"No incident reports were found in {INCIDENTS_DIR}"
        )

    return [parse_incident(path) for path in paths]


def ingest_incidents() -> None:
    incidents = load_incidents()

    print(f"Found {len(incidents)} incident reports.")

    payload = {
        "app_knowledge": incidents,
    }

    response = client.context.ingest(
        database_id=DATABASE_ID,
        request=payload,
    )

    print("Ingestion submitted successfully.")
    print(response)


if __name__ == "__main__":
    ingest_incidents()
```

> The exact request parameter name can vary between HydraDB SDK versions. If your installed SDK expects `body=payload` instead of `request=payload`, change only the `client.context.ingest(...)` call. Keep the document structure unchanged.

Run the ingestion script:

```bash
python3 -m phase1.ingest_incidents
```

Expected result:

```text
Found 3 incident reports.
Ingestion submitted successfully.
...
```

## Step 3 - Verify That Each Incident Was Indexed

Create `phase1/verify.py`:

```python
from __future__ import annotations

import time
from pathlib import Path

from config import DATABASE_ID, client


PROJECT_ROOT = Path(__file__).resolve().parents[1]
INCIDENTS_DIR = PROJECT_ROOT / "data" / "incidents"


def search_for_incident(incident_id: str):
    """
    Search HydraDB for a specific incident identifier.
    """
    return client.search(
        database_id=DATABASE_ID,
        query=incident_id,
        max_results=5,
    )


def verify_incident(incident_id: str, attempts: int = 10) -> bool:
    """
    Poll search until the incident becomes available or the retry limit is hit.
    """
    for attempt in range(1, attempts + 1):
        results = search_for_incident(incident_id)

        if results:
            print(f"✓ {incident_id} is searchable")
            return True

        print(
            f"Waiting for {incident_id} to be indexed "
            f"({attempt}/{attempts})..."
        )
        time.sleep(3)

    print(f"✗ {incident_id} was not found")
    return False


def main() -> None:
    incident_ids = [path.stem for path in sorted(INCIDENTS_DIR.glob("*.md"))]

    if not incident_ids:
        raise FileNotFoundError("No incident files were found.")

    verified = [
        verify_incident(incident_id)
        for incident_id in incident_ids
    ]

    if all(verified):
        print("\nAll incident reports were indexed successfully.")
    else:
        raise RuntimeError(
            "One or more incident reports could not be verified."
        )


if __name__ == "__main__":
    main()
```

Run it:

```bash
python3 -m phase1.verify
```

Indexing may take a few seconds. The script retries automatically.

Expected output:

```text
✓ INC-1001 is searchable
✓ INC-1002 is searchable
✓ INC-1003 is searchable

All incident reports were indexed successfully.
```

## Step 4 - Search Historical Incidents

Create `phase1/search_incidents.py`:

```python
from __future__ import annotations

from typing import Any

from config import (
    DATABASE_ID,
    RECALL_ALPHA,
    RECALL_MAX_RESULTS,
    RECALL_MIN_SCORE,
    client,
)


def get_value(item: Any, field_name: str, default=None):
    """
    Read a value from either an SDK object or a dictionary response.
    """
    if isinstance(item, dict):
        return item.get(field_name, default)

    return getattr(item, field_name, default)


def search_incidents(query: str):
    """
    Search the incident knowledge base using a natural-language question.
    """
    return client.search(
        database_id=DATABASE_ID,
        query=query,
        max_results=RECALL_MAX_RESULTS,
        min_score=RECALL_MIN_SCORE,
        alpha=RECALL_ALPHA,
    )


def print_results(results) -> None:
    if not results:
        print("No relevant incident evidence was found.")
        return

    for index, result in enumerate(results, start=1):
        content = get_value(result, "chunk_content", "")
        score = get_value(result, "relevancy_score", 0.0)
        metadata = get_value(result, "additional_metadata", {}) or {}
        source_title = get_value(result, "source_title", "unknown")

        if not isinstance(metadata, dict):
            metadata = {}

        incident_id = metadata.get("incident_id", source_title)
        service = metadata.get("service", "unknown")
        severity = metadata.get("severity", "unknown")
        source_path = metadata.get("source_path", "unknown")

        print("=" * 80)
        print(f"Result: {index}")
        print(f"Incident: {incident_id}")
        print(f"Service: {service}")
        print(f"Severity: {severity}")
        print(f"Score: {score:.4f}")
        print(f"Source: {source_path}")
        print("-" * 80)
        print(content.strip())
        print()


def main() -> None:
    query = input("Ask an incident-response question: ").strip()

    if not query:
        raise ValueError("The search query cannot be empty.")

    results = search_incidents(query)
    print_results(results)


if __name__ == "__main__":
    main()
```

Run the search:

```bash

```



## Step 5 - Optional Metadata Filtering

Natural-language retrieval finds semantically relevant evidence. Metadata filtering narrows the search to a particular service, severity, date, or incident status.

Add the following function to `phase1/search_incidents.py` if your HydraDB SDK version supports search filters:

```python
def search_high_severity_incidents(query: str):
    return client.search(
        database_id=DATABASE_ID,
        query=query,
        max_results=RECALL_MAX_RESULTS,
        min_score=RECALL_MIN_SCORE,
        alpha=RECALL_ALPHA,
        filters={
            "severity": {
                "$in": ["SEV-1", "SEV-2"],
            }
        },
    )
```

Example:

```python
results = search_high_severity_incidents(
    "incidents caused by invalid configuration"
)
print_results(results)
```

> Filter syntax may differ between SDK releases. Keep metadata fields simple and consistent so they can be mapped easily to the filter syntax supported by your installed version.

## Step 6 - Prevent Duplicate Incident Documents

Stable IDs are important because incident reports may be updated after the initial investigation.

This phase uses IDs such as:

```text
incident/INC-1001
incident/INC-1002
incident/INC-1003
```

When an incident report changes, ingest the document again using the same ID. Depending on your HydraDB database configuration, the existing document can be updated or replaced rather than creating an unrelated duplicate.

Do not generate random IDs for operational documents that have permanent identifiers.

Recommended ID conventions:

```text
incident/{incident_id}
runbook/{service}/{runbook_name}
postmortem/{incident_id}
alert/{monitoring_system}/{alert_name}
service/{service_name}
deployment/{repository}/{commit_sha}
```

These IDs will become important in later phases when the agent connects incidents to runbooks, alerts, deployments, and postmortems.

## Step 2 - Build Investigation API

Create `phase3/app.py`

```python
from fastapi import FastAPI
from openai import OpenAI

from config import (
    DATABASE_ID,
    OPENAI_API_KEY,
    client,
)

llm = OpenAI(api_key=OPENAI_API_KEY)

app = FastAPI()


@app.post("/investigate")
def investigate(question: str):

    results = client.search(
        database_id=DATABASE_ID,
        query=question,
        max_results=10,
    )

    context = "\n\n".join(
        r.chunk_content for r in results
    )

    prompt = f"""
Use only the evidence below.

Evidence:

{context}

Question:

{question}
"""

    response = llm.chat.completions.create(

    }
```
