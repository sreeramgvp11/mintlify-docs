---
title: "List"
description: "Retrieve paginated memories or knowledge for a specific tenant and subtenant combination (POST /list/data)"
openapi: "POST /list/data"
---

Use **POST /list/data** with a JSON body to list either knowledge sources or user memories. Set `kind` to `"knowledge"` or `"memories"` and include `tenant_id` (and optionally `sub_tenant_id`).

All responses are **paginated**. Control the page with `page` and `page_size` (max 100, default 50). Every response includes a `pagination` object with `total`, `total_pages`, `has_next`, and `has_previous` to help you navigate.

### List memories

<Tabs>
  <Tab title="API Request">
    ```bash
    curl -X 'POST' \
    'https://api.hydradb.com/list/data' \
    -H 'accept: application/json' \
    -H 'Content-Type: application/json' \
    -d '{
      "tenant_id": "tenant_123",
      "sub_tenant_id": "sub_tenant_456",
      "kind": "memories",
      "page": 1,
      "page_size": 20
    }'
    ```

### Filtering

Use the optional `filters` object to narrow results by `tenant_metadata`, `document_metadata`, or top-level `source_fields` (`title`, `type`, `description`, `url`, `timestamp`). All provided key-value pairs use **AND** logic.

<Tabs>
  <Tab title="API Request">
    ```bash
    curl -X 'POST' \
    'https://api.hydradb.com/list/data' \
    -H 'accept: application/json' \
    -H 'Content-Type: application/json' \
    -d '{
      "tenant_id": "tenant_123",
      "kind": "knowledge",
      "page": 1,
      "page_size": 20,
      "filters": {
        "tenant_metadata": {"department": "Finance"},
        "document_metadata": {"author": "Alice Smith"},
        "source_fields": {"type": "slack"}
      }
    }'
    ```
  </Tab>
</Tabs>

### Field projection with `include_fields`

Pass `include_fields` to return only the specified fields per source (plus `id`, `tenant_id`, `sub_tenant_id` which are always included). This reduces payload size and improves performance. Allowed values: `attachments`, `content`, `description`, `document_metadata`, `meta`, `note`, `relations`, `tenant_metadata`, `timestamp`, `title`, `type`, `url`.

<Note>Only applies to `kind: "knowledge"`. Ignored for `kind: "memories"`.</Note>

<Tabs>
  <Tab title="API Request">
    ```bash
    curl -X 'POST' \
    'https://api.hydradb.com/list/data' \
    -H 'accept: application/json' \
    -H 'Content-Type: application/json' \
    -d '{
      "tenant_id": "tenant_123",
      "kind": "knowledge",
      "page": 1,
      "page_size": 50,
      "include_fields": ["title", "type", "url", "timestamp"]
    }'
    ```
  </Tab>
</Tabs>

### Fetch specific sources by ID

Pass `source_ids` (max 100) to retrieve a specific set of sources. Pagination and filters still apply.

<Tabs>
  <Tab title="API Request">
    ```bash
    curl -X 'POST' \
    'https://api.hydradb.com/list/data' \
    -H 'accept: application/json' \
    -H 'Content-Type: application/json' \
    -d '{
      "tenant_id": "tenant_123",
      "kind": "knowledge",
      "source_ids": ["src_001", "src_002", "src_003"]
    }'
    ```
  </Tab>
</Tabs>

### Combining filters and field projection

Filters and `include_fields` are independent  -  filtering operates on the full stored document in MongoDB, while projection controls which fields appear in the response. You can filter by a field without including it in the output.

<Tabs>
  <Tab title="API Request">
    ```bash
    curl -X 'POST' \
    'https://api.hydradb.com/list/data' \
    -H 'accept: application/json' \
    -H 'Content-Type: application/json' \
    -d '{
      "tenant_id": "tenant_123",
      "kind": "knowledge",
      "page": 1,
      "page_size": 25,
      "filters": {
        "tenant_metadata": {"department": "Finance"}
      },
      "include_fields": ["title", "type", "timestamp"]
    }'
    ```
  </Tab>
</Tabs>
