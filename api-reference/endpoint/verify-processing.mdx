    ```bash expandable
    # Multiple files
    curl -X POST \
      'https://api.hydradb.com/ingestion/verify_processing?file_ids=file_123&file_ids=file_456&tenant_id=tenant_1234' \
      -H 'Authorization: Bearer YOUR_API_KEY'
    ```

## Status Values

| Status           | Description                             |
| ---------------- | --------------------------------------- |
| `queued`         | Content is waiting to be processed      |
| `processing`     | Content is being chunked and embedded   |
| `graph_creation` | Knowledge graph is being built          |
| `completed`      | Content is fully indexed and searchable |
| `errored`        | Processing failed (check `error_code`)  |

## Response Format

```json
{
  "statuses": [
    {
      "file_id": "file_123",
      "indexing_status": "completed",
      "error_code": "",
      "success": true,
      "message": "Processing status retrieved successfully"
    }
  ]
}
```

<Note>
  The `file_id` parameter is deprecated. Use `file_ids` for both single and batch lookups.
</Note>
