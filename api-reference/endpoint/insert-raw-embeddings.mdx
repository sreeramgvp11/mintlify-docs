## When to use it

Use this endpoint to insert vectors generated by your own embedding pipeline – fine-tuned models, custom architectures, or vectors migrated from another store.

This endpoint requires an **embeddings tenant**. See [Embeddings overview](/api-reference/endpoint/embeddings-overview) for setup.

## Endpoint

- **Auth:** Bearer token
- **Idempotency:** Controlled by `upsert` (default `false` – fails on existing chunks)
- **Async:** No

## Example

<CodeGroup>

## Request parameters

| Name | Type | Required | Default | Description |
|---|---|---|---|---|
| `tenant_id` | string | Yes | – | The embeddings tenant. |
| `embeddings` | array | Yes | – | Array of source documents. See [Embedding document](#embedding-document). |
| `sub_tenant_id` | string | No | default | Sub-tenant scope. |
| `upsert` | boolean | No | `false` | If `true`, existing chunks with the same `chunk_id` are updated. |

## Embedding document

Each entry in `embeddings` represents one source with one or more chunk vectors:

| Field | Type | Required | Description |
|---|---|---|---|
| `source_id` | string | Yes | Identifier for the source. |
| `metadata` | object | No | Source-level metadata. |
| `embeddings` | array | Yes | One or more chunk vectors. |
| `embeddings[].chunk_id` | string | Yes | Unique identifier for the chunk. |
| `embeddings[].embedding` | number[] | Yes | The vector. Length must match `embeddings_dimension` configured at tenant creation. |

## Response

```json
{
  "insert_count": 2,
  "ids": ["doc_001_chunk_0", "doc_001_chunk_1"],
  "success": true,
  "error": null
}
```

| Field | Description |
|---|---|
| `insert_count` | Number of chunks inserted. |
| `ids` | The `chunk_id`s that were inserted. |
| `success` | `true` if the operation succeeded. |
| `error` | Error message if `success: false`. |

## Behavior notes

<Info>
**Vector dimension is fixed.** All vectors must match the `embeddings_dimension` set when the tenant was created. Sending a vector of a different length returns `422 VALIDATION_ERROR`.
</Info>

<Info>
**Default `upsert` is `false`.** Unlike the standard ingestion endpoints, embeddings inserts default to **fail** on existing chunks. Set `upsert: true` to overwrite.
</Info>
