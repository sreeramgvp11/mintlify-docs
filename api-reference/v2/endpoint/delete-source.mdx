import { Field } from "/snippets/field.jsx";

Specify the resource category with the `type` parameter:

- `type=knowledge` _(default)_  -  delete knowledge sources.
- `type=memory`  -  delete memories.

Pass one or more IDs in `request.ids`. The body wraps two fields together: `request` (the actual delete payload) and `type` (the bucket).

### Knowledge deletion 

Use `type: "knowledge"` and pass knowledge `ids` in the `ids` array. Include the same `sub_tenant_id` you used when ingesting the knowledge; omitting it targets the default sub-tenant.

<RequestExample>

```python Python SDK
client.context.delete(
    type="knowledge",
    tenant_id="acme_corp",
    sub_tenant_id="team_docs",
    ids=["policy_main", "runbook_deploy"],
)
```

```typescript TypeScript SDK
await client.context.delete({
  type: "knowledge",
  tenantId: "acme_corp",
  subTenantId: "team_docs",
  ids: ["policy_main", "runbook_deploy"],
});
```

```bash cURL
curl -X DELETE 'https://api.hydradb.com/context' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2" \
  -H "Content-Type: application/json" \
  -d '{
    "request": {
      "tenant_id": "acme_corp",
      "sub_tenant_id": "team_docs",
      "ids": ["policy_main", "runbook_deploy"]
    },
    "type": "knowledge"
  }'
```

</RequestExample>

### Memory deletion 

Use `type: "memory"` and pass memory `ids` in the `ids` array. Include the same `sub_tenant_id` you used when ingesting the memories; omitting it targets the default sub-tenant.

<RequestExample>

```python Python SDK
client.context.delete(
    type="memory",
    tenant_id="acme_corp",
    sub_tenant_id="user_alex",
    ids=["mem_user_alex_tone"],
)
```

```typescript TypeScript SDK
await client.context.delete({
  type: "memory",
  tenantId: "acme_corp",
  subTenantId: "user_alex",
  ids: ["mem_user_alex_tone"],
});
```

```bash cURL
curl -X DELETE 'https://api.hydradb.com/context' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2" \
  -H "Content-Type: application/json" \
  -d '{
    "request": {
      "tenant_id": "acme_corp",
      "sub_tenant_id": "user_alex",
      "ids": ["mem_user_alex_tone"]
    },
    "type": "memory"
  }'
```

</RequestExample>

<ResponseExample>

```json Success
{
  "success": true,
  "data": {
    "success": true,
    "user_memory_deleted": true
  },
  "error": null,
  "meta": {
    "request_id": "9d13aef4-02f4-4e73-8c62-4c2601d04f9d",
    "latency_ms": 12.3
  }
}
```

```json Failure
{
  "success": false,
  "data": null,
  "error": {
    "code": "SOURCE_NOT_FOUND",
    "message": "Source not found"
  },
  "meta": {
    "request_id": "9d13aef4-02f4-4e73-8c62-4c2601d04f9d",
    "latency_ms": 4.8
  }
}
```

</ResponseExample>

## Some additional notes

- **Partial-success semantics:** For `type=knowledge`, each ID is reported independently in `results[]`. A failure on one ID does not stop the rest. For `type=memory`, the response reports an aggregate `user_memory_deleted` reflecting _all_ listed IDs.
- **Retrieval drops the source immediately:** Even before background cleanup finishes, deleted IDs disappear from `/query` and `/context/list` responses.
- **Mixed deletes need two calls:** To delete both knowledge and memory items, send two requests  -  one with `type=knowledge`, one with `type=memory`.
