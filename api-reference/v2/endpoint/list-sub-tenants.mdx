Sub-tenants are created implicitly during ingestion the first time a `sub_tenant_id` value is written. For details, see the [Multi-Tenant Support](/essentials/v2/multi-tenant) guide.

## Behavior notes

<Info>
**Implicit creation.** Sub-tenants are auto-created when ingestion writes data under a new `sub_tenant_id`. The returned list grows organically as your application writes data under new values.
</Info>

- **Default sub-tenant included:** HydraDB assigns an auto-generated default sub-tenant ID for calls that omit `sub_tenant_id`. That default is created once and reused. It appears here once any data is written without an explicit `sub_tenant_id`.
- **Only sub-tenants with data are listed:** A sub-tenant that has been referenced in a query but never received an ingest will not show up here.

Common codes: `404 TENANT_NOT_FOUND`, `401 UNAUTHORIZED`, `422 VALIDATION_ERROR`. See [Error Responses](/api-reference/v2/error-responses) for the full list.
