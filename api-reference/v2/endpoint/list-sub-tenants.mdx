Retrieve all sub-tenant IDs that have indexed data within a specific tenant.

Sub-tenants are created implicitly during ingestion the first time a `sub_tenant_id` value is written. For details, see the [Multi-Tenant Support](/essentials/v2/multi-tenant) guide.

<RequestExample>
```bash cURL
curl -X GET 'https://api.hydradb.com/tenants/sub-tenants?tenant_id=my_first_tenant' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2"
```
```typescript TypeScript SDK
const response = await client.tenants.subTenants({
  tenant_id: "my_first_tenant",
});
```
```python Python SDK
response = client.tenants.sub_tenants(tenant_id="my_first_tenant")
```
</RequestExample>



## Query parameters



| Name | Description |
| --- | --- |
| <Field name="tenant_id" type="string" required /> | Identifier of the tenant whose sub-tenants you want to list. |

## Response

<ResponseExample>

```json Response
{
  "sub_tenant_ids": ["user_alex", "user_johndoe", "workspace_42", "subtenant_default_abc123"],
  "message": "Successfully retrieved sub-tenant IDs"
}
```

</ResponseExample>

A tenant with no indexed content can return an empty list:

```json Empty tenant
{
  "sub_tenant_ids": [],
  "message": "Successfully retrieved sub-tenant IDs"
}
```

| Field | Description |
|---|---|
| <Field name="sub_tenant_ids" type="string[]" /> | Array of unique sub-tenant IDs that have indexed data in either collection (Knowledge or Memories). |
| <Field name="message" type="string" /> | Human-readable status (string). |

## Behavior notes

<Info>
**Implicit creation.** Sub-tenants are auto-created when ingestion writes data under a new `sub_tenant_id`. The returned list grows organically as your application writes data under new values.
</Info>

- **Default sub-tenant included:** HydraDB assigns an auto-generated default sub-tenant ID for calls that omit `sub_tenant_id`. That default is created once and reused. It appears here once any data is written without an explicit `sub_tenant_id`.
- **Only sub-tenants with data are listed:** A sub-tenant that has been referenced in a search but never received an ingest will not show up here.
