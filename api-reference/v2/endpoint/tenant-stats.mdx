Get detailed metrics including storage growth, object counts, and vector dimensions.

The response splits stats into two tenant-wide collections: `normal_collection` (Knowledge) and `memory_collection` (Memories), each with its own row count and embedding dimensionality. Counts aggregate across all sub-tenants in the tenant.

<RequestExample>
```bash cURL
curl -X GET 'https://api.hydradb.com/tenants/stats?tenant_id=my_first_tenant' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2"
```
```typescript TypeScript SDK
const stats = await client.tenants.stats({
  tenant_id: "my_first_tenant",
});
```
```python Python SDK
stats = client.tenants.stats(tenant_id="my_first_tenant")
```
</RequestExample>



## Query parameters



| Name | Description |
| --- | --- |
| <Field name="tenant_id" type="string" required /> | Identifier of the tenant to report on. |

<ResponseExample>

```json Response
{
  "tenant_id": "my_first_tenant",
  "normal_collection": {
    "row_count": 42318,
    "dimensions": 1536
  },
  "memory_collection": {
    "row_count": 287,
    "dimensions": 1536
  },
  "message": "Successfully retrieved tenant collection statistics"
}
```

</ResponseExample>

For a new or empty tenant, both collections are present and report zero rows:

```json Empty tenant
{
  "tenant_id": "my_first_tenant",
  "normal_collection": { "row_count": 0, "dimensions": 1536 },
  "memory_collection": { "row_count": 0, "dimensions": 1536 },
  "message": "Successfully retrieved tenant collection statistics"
}
```

## Behavior notes

<Info>
**`row_count` is chunks, not sources.** One ingested document typically becomes many chunks (e.g., a 30-page PDF can produce 100+ rows). To count distinct sources, use [List Documents](/api-reference/v2/endpoint/list-documents) with `page_size=1` and read `total`.
</Info>

- **Both collections always exist:** Even if you've only ingested knowledge (or only memories), both collections are provisioned. Empty collections report `row_count: 0` with the configured `dimensions`.
- **Stats are eventually consistent:** Immediately after ingestion or deletion, counts may lag until background processing completes. Check [Source Status](/api-reference/v2/endpoint/source-status) for recently ingested source IDs before treating counts as final.
