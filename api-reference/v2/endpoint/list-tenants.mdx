---
title: "List Tenants"
description: "List all tenants created. "
openapi: "api-reference/v2/openapi.json GET /tenants"
---

The response separates active or provisioning tenants (in `data.tenant_ids`) from tenants whose provisioning failed (in `data.failed_tenant_ids`). Use [Tenant Status](https://docs.hydradb.com/api-reference/v2/endpoint/tenant-status) to confirm readiness before ingestion.

<RequestExample>

```python Python SDK
response = client.tenants.list()
```

```typescript TypeScript SDK
const response = await client.tenants.list();
```

```bash cURL
curl -X GET 'https://api.hydradb.com/tenants' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2"
```

## Retry notes

- If provisioning failed for a tenant, `data.failed_tenant_ids` contains diagnostic entries as shown in the **Provisioning issue** tab.
- **Retry failed tenants:** If a tenant appears in `data.failed_tenant_ids`, re-create that tenant with `POST /tenants` after addressing the reported issue. Poll status again before ingestion.
