---
title: "List Tenants"
description: "List all tenant IDs registered for your organization."
openapi: "api-reference/v2/openapi.json GET /tenants"
---

import { Field } from "/snippets/field.jsx";

The response separates active or provisioning tenants (in `tenant_ids`) from tenants whose provisioning failed (in `failed_tenant_ids`). 

<RequestExample>
```python Python SDK
response = client.tenants.list()
```
```typescript TypeScript SDK
const response = await client.tenants.list();
```
```bash cURL
curl -X GET 'https://api.hydradb.com/tenants' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2"
```
</RequestExample>



<ResponseExample>

```json Success
{
  "tenant_ids": ["acme_corp", "my_first_tenant"],
  "failed_tenant_ids": null,
  "message": "Successfully retrieved tenant IDs"
}
```

```json Provisioning Failure
{
  "tenant_ids": ["acme_corp"],
  "failed_tenant_ids": [
    {
      "tenant_id": "staging_import",
      "error": "Provisioning failed. Re-create the tenant to retry."
    }
  ],
  "message": "Successfully retrieved tenant IDs"
}
```

</ResponseExample>

If provisioning failed for a tenant, `failed_tenant_ids` contains diagnostic entries as shown in the **Provisioning Failure** tab.

<Info>
**Includes provisioning tenants.** Tenants returned in `tenant_ids` may still be provisioning. Use [Tenant Status](/api-reference/v2/endpoint/tenant-status) to confirm readiness before ingestion.
</Info>

## Behavior notes
- **Retry failed tenants:** If a tenant appears in `failed_tenant_ids`, re-create that tenant with `POST /tenants` after addressing the reported issue. Poll status again before ingestion.

## Errors

Common codes: `401 UNAUTHORIZED`, `500 INTERNAL_ERROR`. See [Error Responses](/api-reference/v2/error-responses) for the full list.
