## When to use it

The single ingestion endpoint for everything that flows into HydraDB. Pick `type` to choose the bucket, then pass the matching content fields:

- **`type=knowledge`**  -  shared content. Use `files` for binary uploads HydraDB should parse (PDF, DOCX, CSV, MD, TXT), or `app_knowledge` for pre-extracted JSON content (Slack threads, Notion pages, web pages). Both can be sent in a single request.
- **`type=memory`**  -  per-user content. Scope with `sub_tenant_id`. Use `infer: true` to let HydraDB extract preferences from raw signal automatically; `infer: false` stores the text verbatim.

Sends `multipart/form-data`. Nested JSON fields (`file_metadata`, `app_knowledge`, `memories`) must be sent as **JSON-stringified** form values.

## Endpoint

- **Auth:** Bearer token + `API-Version: 2`
- **Idempotency:** Controlled by `upsert`  -  `true` (default) replaces an existing source with the same ID; `false` errors on conflict.
- **Async:** Yes  -  returns `202 Accepted` with the new `source_id`s. Poll [Source Status](/api-reference/v2/endpoint/source-status) until each item reports `completed`.
- **Content-Type:** `multipart/form-data` (not JSON).

## Examples

### Knowledge  -  files

<RequestExample>

```bash cURL
curl -X POST 'https://api.hydradb.com/source/ingest' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2" \
  -F "type=knowledge" \
  -F "tenant_id=acme_corp" \
  -F "files=@/path/to/policy.pdf" \
  -F "files=@/path/to/runbook.pdf" \
  -F 'file_metadata=[
    { "file_id": "policy_v2", "tenant_metadata": { "department": "legal" } },
    { "file_id": "runbook_deploy", "tenant_metadata": { "department": "ops" } }
  ]'
```

```typescript TypeScript SDK
const result = await client.source.ingest({
  type: "knowledge",
  tenant_id: "acme_corp",
  files: [
    { path: "/path/to/policy.pdf", filename: "policy.pdf", contentType: "application/pdf" },
    { path: "/path/to/runbook.pdf", filename: "runbook.pdf", contentType: "application/pdf" },
  ],
  file_metadata: JSON.stringify([
    { file_id: "policy_v2", tenant_metadata: { department: "legal" } },
    { file_id: "runbook_deploy", tenant_metadata: { department: "ops" } },
  ]),
});
```

```python Python SDK
import json

with open("/path/to/policy.pdf", "rb") as f1, open("/path/to/runbook.pdf", "rb") as f2:
    result = client.source.ingest(
        type="knowledge",
        tenant_id="acme_corp",
        files=[
            ("policy.pdf", f1, "application/pdf"),
            ("runbook.pdf", f2, "application/pdf"),
        ],
        file_metadata=json.dumps([
            {"file_id": "policy_v2", "tenant_metadata": {"department": "legal"}},
            {"file_id": "runbook_deploy", "tenant_metadata": {"department": "ops"}},
        ]),
    )
```

</RequestExample>

## Form fields

Required fields are marked with `*`.

| Name | Type | Description |
|---|---|---|
| <Field name="type" /> | `"knowledge" \| "memory"` | Bucket selector. (default=`"knowledge"`) |
| <Field name="tenant_id" required /> | string | Target tenant. |
| <Field name="sub_tenant_id" /> | string | Logical partition inside the tenant. (default=`""`  -  default sub-tenant) |
| <Field name="upsert" /> | boolean | Replace existing sources with the same ID. Set to `false` to error on conflict. (default=`true`) |
| <Field name="files" /> | file[] | Binary uploads, **knowledge only**. Required when `type=knowledge` and you have files to upload. Omit when ingesting memories. (default=`[]`) |
| <Field name="file_metadata" /> | string (JSON array) | One entry per file in `files`, same order. See [`file_metadata` item shape](#file_metadata-item-shape). |
| <Field name="app_knowledge" /> | string (JSON object or array) | Pre-extracted source objects (Slack, Notion, web pages, etc.), **knowledge only**. See [`app_knowledge` item shape](#app_knowledge-item-shape). |
| <Field name="memory" /> | string (JSON array) | Memory items, **memory only**. Required when `type=memory`. See [`memories` item shape](#memories-item-shape). |

### `file_metadata` item shape

| Field | Type | Description |
|---|---|---|
| <Field name="file_id" /> | string | Optional source ID. If set, becomes the `source_id` for this file (use your app's document ID for parity). |
| <Field name="tenant_metadata" /> | object | Tenant-schema fields for filtering and search. Keys must be declared in `tenant_metadata_schema`. (default=`{}`) |
| <Field name="document_metadata" /> | object | Free-form per-document fields for display or bookkeeping. To filter on these at search time, nest under `metadata_filters.document_metadata`. (default=`{}`) |
| <Field name="title" /> | string | Override the source title shown in search results. (default=filename) |
| <Field name="type" /> | string | Override the source type shown in search results. (default=inferred) |
| <Field name="url" /> | string | Override the canonical URL. |
| <Field name="timestamp" /> | string | ISO-8601 timestamp override. (default=upload time) |
| <Field name="relations" /> | object | Declare forceful relations to other sources. Shape: `{ "cortex_source_ids": ["...", "..."] }`. Surfaced via `additional_context` in `mode: "thinking"` search. |

### `app_knowledge` item shape

| Field | Type | Description |
|---|---|---|
| <Field name="id" /> | string | Source ID. Treated as the upsert key. (default=auto-generated) |
| <Field name="title" /> | string | Short title or subject shown in search results. |
| <Field name="type" /> | string | Source category (`slack`, `notion`, `gmail`, `webpage`, etc.). Used for filtering and display. |
| <Field name="description" /> | string | Optional long-form description. |
| <Field name="url" /> | string | Canonical URL or reference link. |
| <Field name="timestamp" /> | string | ISO-8601 timestamp (creation or last-updated). |
| <Field name="content" /> | object | Content payload. Use `{ "text": "..." }` for plain text. |
| <Field name="tenant_metadata" /> | object | Tenant-schema fields. (default=`{}`) |
| <Field name="document_metadata" /> | object | Free-form per-document fields. (default=`{}`) |
| <Field name="attachments" /> | array | Optional related attachments. (default=`[]`) |
| <Field name="relations" /> | object | Forceful relations, same shape as on `file_metadata`. |

### `memories` item shape

| Field | Type | Description |
|---|---|---|
| <Field name="source_id" /> | string | Optional unique ID. Acts as the upsert key. (default=auto-generated) |
| <Field name="title" /> | string | Short label for display in `/source/list` and search hits as `source.title`. (default=truncated `text`) |
| <Field name="text" /> | string | Raw text or markdown content. Use this **or** `user_assistant_pairs`. |
| <Field name="user_assistant_pairs" /> | array | Conversation pairs `{ user, assistant }`. Use this **or** `text`. |
| <Field name="is_markdown" /> | boolean | Treat `text` as markdown for chunking. (default=`false`) |
| <Field name="infer" /> | boolean | When `true`, HydraDB extracts the underlying preference from raw signal. (default=`false`) |
| <Field name="custom_instructions" /> | string | Guides extraction when `infer: true`. Ignored when `infer: false`. |
| <Field name="user_name" /> | string | The user's name. Feeds inference. (default=`"User"`) |
| <Field name="expiry_time" /> | integer | TTL in seconds. Memory stops surfacing after expiry. |
| <Field name="tenant_metadata" /> | object | Tenant-schema fields. (default=`{}`) |
| <Field name="document_metadata" /> | object | Free-form per-document fields. (default=`{}`) |
| <Field name="relations" /> | object | Forceful relations within the Memories store. Shape: `{ "cortex_source_ids": ["...", "..."] }`. |

## Response

<ResponseExample>

```json Response
{
  "success": true,
  "message": "Knowledge uploaded successfully",
  "results": [
    { "source_id": "policy_v2", "filename": "policy.pdf", "status": "queued", "error": null },
    { "source_id": "runbook_deploy", "filename": "runbook.pdf", "status": "queued", "error": null }
  ],
  "success_count": 2,
  "failed_count": 0
}
```

</ResponseExample>

| Field | Description |
|---|---|
| <Field name="success" /> | Overall ingest acceptance  -  `true` if at least one item was queued. |
| <Field name="results[]" /> | One entry per queued item with its `source_id` and initial `status` of `queued`. |
| <Field name="results[].source_id" /> | Use this for status polling, fetch, list, delete, and graph relations. |
| <Field name="results[].error" /> | Non-null only if HydraDB rejected the item before queueing (e.g., schema validation). |
| <Field name="success_count / failed_count" /> | Tallies. |

## Behavior notes

<Info>
**Async indexing.** `202 Accepted` only means HydraDB **queued** the work. Content is not retrievable until `indexing_status` reaches `graph_creation` (partial) or `completed` (full). Always poll [Source Status](/api-reference/v2/endpoint/source-status) before searching.
</Info>

<Info>
**`type=knowledge` + `app_knowledge` and `files` mix.** You can send both in the same request. HydraDB processes each item independently and returns one `result` per source.
</Info>

<Info>
**Multipart, not JSON.** This is one of the few endpoints that uses `multipart/form-data`. Stringify the JSON arrays (`file_metadata`, `app_knowledge`, `memories`) before sending them in the form field.
</Info>

<Warning>
**`tenant_metadata` keys must be declared.** Keys not in the tenant's `tenant_metadata_schema` are silently ignored at search time when used in `metadata_filters`. Plan schema fields up front via [Create Tenant](/api-reference/v2/endpoint/create-tenant).
</Warning>
