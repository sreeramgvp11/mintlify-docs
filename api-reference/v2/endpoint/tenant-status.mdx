Tenant creation is asynchronous. Always poll this endpoint after `POST /tenants` and before your first ingestion or search call  -  the tenant is only ready when **all** of these are `true`:

- Both `infra.vectorstore_status.knowledge` and `infra.vectorstore_status.memories` are `true`
- `infra.graph_status` (the knowledge-graph layer)
- `infra.scheduler_status` (the background indexing scheduler)

<Note>
Provisioning typically takes 1–2 minutes on Free and Ship plans, and 4–5 minutes on Enterprise plans, which get complete physical isolation.
</Note>

<RequestExample>
```bash cURL
curl -X GET 'https://api.hydradb.com/tenants/status?tenant_id=my_first_tenant' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2"
```
```typescript TypeScript SDK
const status = await client.tenants.status({
  tenant_id: "my_first_tenant",
});
```
```python Python SDK
status = client.tenants.status(tenant_id="my_first_tenant")
```
</RequestExample>

```json Response

  "tenant_id": "my_first_tenant",

  "infra": {
    "scheduler_status": true,

      "knowledge": true,
      "memories": true
    }

  "message": "Deployed infrastructure status"

```

</ResponseExample>

| Field | Description |
|---|---|
| <Field name="tenant_id" /> | The tenant being inspected (string). |
| <Field name="org_id" /> | Owning organization ID (string). |
| <Field name="infra" /> | Nested infrastructure status object. See [Infra fields](#infra-fields). |
| <Field name="message" /> | Human-readable status (string). |

### `Infra` fields

| Field | Type | Description |
|---|---|---|
| <Field name="scheduler_status" /> | boolean | Whether the background indexing scheduler is up. |
| <Field name="graph_status" /> | boolean | Whether the knowledge-graph layer is provisioned. |
| <Field name="vectorstore_status" /> | object | `{ knowledge: boolean, memories: boolean }`  -  provisioning status of each vectorstore collection. See [VectorstoreStatus fields](#vectorstorestatus-fields). |

### `VectorstoreStatus` fields

| Field | Type | Description |
|---|---|---|
| <Field name="knowledge" /> | boolean | Whether the Knowledge collection has been provisioned. |
| <Field name="memories" /> | boolean | Whether the Memories collection has been provisioned. |

<Info>
The tenant is fully ready only when both `vectorstore_status.knowledge` and `vectorstore_status.memories` are `true`.
</Info>

## Polling pattern

Poll every 10–20 seconds. Stop and surface a provisioning issue if the tenant is not ready after the expected window for your plan.

A tenant is ready when:

- `infra.scheduler_status === true`
- `infra.graph_status === true`
- `infra.vectorstore_status[0] === true`
- `infra.vectorstore_status[1] === true`

<CodeGroup>
```typescript TypeScript
while (true) {
  const status = await client.tenants.status({ tenant_id: "acme" });
  const { graph_status, vectorstore_status, scheduler_status } = status.infra;
  if (graph_status && scheduler_status && vectorstore_status.knowledge && vectorstore_status.memories) {
    break;
  }
  await new Promise((r) => setTimeout(r, 15000));
}
```
```python Python
import time

while True:
    status = client.tenants.status(tenant_id="acme")
    infra = status.infra
    if (
        infra.graph_status
        and infra.scheduler_status
        and infra.vectorstore_status.knowledge
        and infra.vectorstore_status.memories
    ):
        break
    time.sleep(15)
```
```bash cURL polling loop
TENANT="acme"
until curl -s -X GET "https://api.hydradb.com/tenants/status?tenant_id=${TENANT}" \
       -H "Authorization: Bearer ${HYDRA_DB_API_KEY}" \
       -H "API-Version: 2" \
     | jq -e '.data.infra | .graph_status and .scheduler_status and .vectorstore_status.knowledge and .vectorstore_status.memories' > /dev/null; do
  sleep 2
done
```
</CodeGroup>

## Behavior notes

<Info>
**Eventual consistency.** A freshly created tenant may briefly return `false` on one or more flags. Keep polling  -  provisioning is the normal path.
</Info>

<Info>
**Stale tenant IDs.** If `tenant_id` does not exist, the call returns `404 TENANT_NOT_FOUND`. Always check that the tenant was created successfully before polling.
</Info>
