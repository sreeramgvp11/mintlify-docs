Since tenant creation is asynchronous, poll this endpoint after calling `POST /tenants` and before executing ingestion or search requests.

The tenant is fully ready when:
- Both `infra.vectorstore_status.knowledge` and `infra.vectorstore_status.memory` are `true`
- `infra.graph_status` (the knowledge-graph layer) is `true`
- `infra.scheduler_status` (the background indexing scheduler) is `true`

For details on multi-tenant architecture and provisioning guarantees, see the [Multi-Tenant Support](/essentials/v2/multi-tenant) guide.

## Polling pattern

Poll every 10–20 seconds. Stop and surface a provisioning issue if the tenant is not ready after the expected window for your plan.

A tenant is ready when:

- `infra.scheduler_status === true`
- `infra.graph_status === true`
- `infra.vectorstore_status.knowledge === true`
- `infra.vectorstore_status.memory === true`

<CodeGroup>
```python Python
import time

while True:
    status = client.tenants.status(tenant_id="acme")
    infra = status.infra
    if (
        infra.graph_status
        and infra.scheduler_status
        and infra.vectorstore_status.knowledge
        and infra.vectorstore_status.memory
    ):
        break
    time.sleep(15)
```
```typescript TypeScript
while (true) {
  const status = await client.tenants.status({ tenant_id: "acme" });
  const { graph_status, vectorstore_status, scheduler_status } = status.infra;
  if (graph_status && scheduler_status && vectorstore_status.knowledge && vectorstore_status.memory) {
    break;
  }
  await new Promise((r) => setTimeout(r, 15000));
}
```
```bash cURL polling loop
TENANT="acme"
until curl -s -X GET "https://api.hydradb.com/tenants/status?tenant_id=${TENANT}" \
     -H "Authorization: Bearer ${HYDRA_DB_API_KEY}" \
     -H "API-Version: 2" \
     | jq -e '.data.infra | .graph_status and .scheduler_status and .vectorstore_status.knowledge and .vectorstore_status.memory' > /dev/null; do
  sleep 2
done
```
</CodeGroup>

## Behavior notes

- **Eventual consistency:** A freshly created tenant may briefly return `false` on one or more flags. Keep polling  -  provisioning is the normal path.


- **Stale tenant IDs:** If `tenant_id` does not exist, the call returns `404 TENANT_NOT_FOUND`. Always verify that the tenant was created successfully before polling.

## Errors

Common codes: `404 TENANT_NOT_FOUND`, `401 UNAUTHORIZED`, `422 VALIDATION_ERROR`. See [Error Responses](/api-reference/v2/error-responses) for the full list.
