Specify the `source_id` of the knowledge source or memory you want to retrieve.

<RequestExample>

```python Python SDK
file = client.source.fetch(
    source_id="policy_main",
    tenant_id="acme_corp",
    mode="both",
    expiry_seconds=3600,
)
```

```typescript TypeScript SDK
const file = await client.source.fetch({
  source_id: "policy_main",
  tenant_id: "acme_corp",
  mode: "both",
  expiry_seconds: 3600,
});
```

```bash cURL
curl -G 'https://api.hydradb.com/source/fetch' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2" \
  --data-urlencode "source_id=policy_main" \
  --data-urlencode "tenant_id=acme_corp" \
  --data-urlencode "mode=both" \
  --data-urlencode "expiry_seconds=3600"
```

</RequestExample>

## Query parameters

| Name | Description |
| --- | --- |
| <Field name="source_id" type="string" required /> | ID of the source to fetch. |
| <Field name="tenant_id" type="string" required /> | Owning tenant. |
| <Field name="sub_tenant_id" type="string or null" /> | Sub-tenant scope. If omitted, the default sub-tenant is used. (default=`null`) |
| <Field name="mode" type='"content" or "url" or "both"' /> | What to return. See [Fetch modes](#fetch-modes). (default=`"content"`) |
| <Field name="expiry_seconds" type="integer" /> | TTL of the presigned URL (when `mode` includes `url`). Range `60 ≤ x ≤ 604800` (7 days). (default=`3600`) |

## Fetch modes

| Mode | Returns | Use when |
|---|---|---|
| `content` *(default)* | Parsed text (`content` field) or base64 for binary files (`content_base64`). | You want to render the parsed text in-app or feed it to another model. |
| `url` | Presigned URL (`presigned_url`) valid for `expiry_seconds`. | You want a client or a service to download the original file directly without proxying through your backend. |
| `both` | Text **and** presigned URL. | UI flows that show parsed text inline plus a "Download original" link. |

<Info>
Use `mode=url` when you need the original file. Use `mode=content` when you only need extracted text for display, summarization, or prompting.
</Info>

## Behavior notes

<Info>
**Text vs binary handling.** When the source is text-parseable (PDF, DOCX, MD, TXT), `content` is populated with parsed text. For non-text/binary files, HydraDB returns `content_base64`. Check both fields when handling unknown content types.
</Info>

- **Recently ingested sources:** Fetching immediately after ingestion may return a record before the parsed text is ready. For reliable reads, poll [Source Status](/api-reference/v2/endpoint/source-status) for the `source_id` first.
- **Presigned URL TTL:** The URL is valid only for `expiry_seconds`. Anyone with the URL can download the file during that window, so treat it as a short-lived secret.
- **Memory items:** Fetching a memory's `source_id` returns its raw text content. There are no presigned URLs for memory items  -  `mode=url` and `mode=both` return `presigned_url: null`.

## Errors

Common codes: `400 INVALID_PARAMETERS`, `404 SOURCE_NOT_FOUND`, `422 VALIDATION_ERROR`. See [Error Responses](/api-reference/v2/error-responses) for the full list.
