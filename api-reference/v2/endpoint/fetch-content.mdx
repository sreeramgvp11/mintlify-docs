Pull back ingested content for display in your UI, audit trails, or to feed a different downstream model. `content` is parsed text extracted by HydraDB; `presigned_url` downloads the original file when one exists. Common patterns:

- **UI rendering**  -  show the original document next to LLM-generated answers.
- **Audit views**  -  let admins inspect what was actually ingested for a `source_id` returned by search.
- **Re-processing**  -  fetch the raw text to run your own analyzer or summarizer.
- **Long-lived links**  -  get a presigned URL valid up to 7 days for downstream services to access the file directly.

Works for both knowledge sources and memories  -  pass the `source_id` from the ingest response or from a search hit.

<RequestExample>

```bash cURL
curl -G 'https://api.hydradb.com/source/fetch' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2" \
  --data-urlencode "source_id=policy_main" \
  --data-urlencode "tenant_id=acme_corp" \
  --data-urlencode "mode=both" \
  --data-urlencode "expiry_seconds=3600"
```

```typescript TypeScript SDK
const file = await client.source.fetch({
  source_id: "policy_main",
  tenant_id: "acme_corp",
  mode: "both",
  expiry_seconds: 3600,
});
```

```python Python SDK
file = client.source.fetch(
    source_id="policy_main",
    tenant_id="acme_corp",
    mode="both",
    expiry_seconds=3600,
)
```

</RequestExample>

<Accordion title="HTTP request details">

- **Auth:** Bearer token + `API-Version: 2`
- **Idempotency:** Read-only
- **Async:** No
- **Presigned URL lifetime:** `expiry_seconds` is clamped to `60 ≤ x ≤ 604800` (7 days).

</Accordion>

## Query parameters



| Name | Type | Description |
|---|---|---|
| <Field name="source_id" required /> | string | ID of the source to fetch. |
| <Field name="tenant_id" required /> | string | Owning tenant. |
| <Field name="sub_tenant_id" /> | string \| null | Sub-tenant scope. If omitted, the default sub-tenant is used. (default=`null`) |
| <Field name="mode" /> | `"content" \| "url" \| "both"` | What to return. See [Fetch modes](#fetch-modes). (default=`"content"`) |
| <Field name="expiry_seconds" /> | integer | TTL of the presigned URL (when `mode` includes `url`). Range `60 ≤ x ≤ 604800` (7 days). (default=`3600`) |

## Fetch modes

| Mode | Returns | Use when |
|---|---|---|
| `content` *(default)* | Parsed text (`content` field) or base64 for binary files (`content_base64`). | You want to render the parsed text in-app or feed it to another model. |
| `url` | Presigned URL (`presigned_url`) valid for `expiry_seconds`. | You want a client or a service to download the original file directly without proxying through your backend. |
| `both` | Text **and** presigned URL. | UI flows that show parsed text inline plus a "Download original" link. |

<Info>
Use `mode=url` when you need the original file. Use `mode=content` when you only need extracted text for display, summarization, or prompting.
</Info>

## Response

<ResponseExample>

```json Response
{
  "success": true,
  "source_id": "policy_main",
  "content": "Section 1. Authentication policies...",
  "content_base64": null,
  "presigned_url": "https://storage.hydradb.com/.../policy_main.pdf?X-Amz-...",
  "content_type": "application/pdf",
  "size_bytes": 1842233,
  "message": "File fetched successfully"
}
```

</ResponseExample>

| Field | Description |
|---|---|
| <Field name="success" /> | Boolean  -  always `true` on a 200 response. |
| <Field name="source_id" /> | The source you asked for (string). |
| <Field name="content" /> | Parsed text content (string, or `null`). Present when `mode` is `content` or `both` and the file is text-parseable. |
| <Field name="content_base64" /> | Base64-encoded raw file content (string, or `null`). Present when the source is binary and `mode` includes content. |
| <Field name="presigned_url" /> | Direct download URL (string, or `null`). Present when `mode` is `url` or `both`. |
| <Field name="content_type" /> | MIME type when known (string, or `null`). |
| <Field name="size_bytes" /> | Source file size in bytes when known (integer, or `null`). |
| <Field name="message" /> | Human-readable status (string). |

## Behavior notes

<Info>
**Text vs binary.** When the source is text-parseable (PDF, DOCX, MD, TXT), `content` is the parsed text. For non-text files HydraDB returns `content_base64` instead. Check both when handling unknown content types.
</Info>

<Info>
**Recently ingested sources.** If you fetch immediately after ingestion, the source record may exist before parsed text is ready. For reliable reads, poll [Source Status](/api-reference/v2/endpoint/source-status) for that `source_id`; use `mode=url` for the original file and `mode=content` after processing when you need extracted text.
</Info>

<Info>
**Presigned URL TTL.** The URL is valid only for `expiry_seconds`. Anyone with the URL can download the file during that window  -  treat it as a short-lived secret.
</Info>

<Info>
**Memory items.** Fetching a memory's `source_id` returns its raw text content (and the inferred preference, when applicable). There is no presigned URL for memory items  -  `mode=url` and `mode=both` return `presigned_url: null`.
</Info>
