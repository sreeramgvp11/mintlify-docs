## When to use it

The single retrieval endpoint for everything. Use it any time you need to feed an LLM with grounded context, surface user preferences, or fetch chunks ranked by relevance.

Two independent dimensions control behavior:

- **`source`** picks **what** to search: `"sources"` (knowledge), `"memories"` (user-scoped), or `"all"` (both, merged and re-ranked together).
- **`search_by`** picks **how** to match: `"hybrid"` (semantic + BM25, the default), `"text"` (BM25 only — pair with `operator`), or `"vector"` (pure dense vectors).

Use `mode: "thinking"` for higher-quality multi-query retrieval; `"fast"` for low-latency single-pass. Turn on `graph_context: true` to surface entity/relation paths alongside chunks. Tune `alpha`, `recency_bias`, and `metadata_filters` to shape ranking and narrow the candidate set.

## Endpoint

- **Auth:** Bearer token + `API-Version: 2`
- **Idempotency:** Read-only
- **Async:** No

## Examples

### Document Q&A

<CodeGroup>
```bash cURL
curl -X POST 'https://api.hydradb.com/search' \
  -H "Authorization: Bearer <your_api_key>" \
  -H "API-Version: 2" \
  -H "Content-Type: application/json" \
  -d '{
    "tenant_id": "acme_corp",
    "query": "What is our refund policy?",
    "source": "sources",
    "search_by": "hybrid",
    "mode": "thinking",
    "graph_context": true
  }'
```
```typescript TypeScript SDK
const result = await client.search.query({
  tenant_id: "acme_corp",
  query: "What is our refund policy?",
  source: "sources",
  search_by: "hybrid",
  mode: "thinking",
  graph_context: true,
});
```
```python Python SDK
result = client.search.query(
    tenant_id="acme_corp",
    query="What is our refund policy?",
    source="sources",
    search_by="hybrid",
    mode="thinking",
    graph_context=True,
)
```
</CodeGroup>

## Request body

| Name | Type / values | Required | Default | Description |
|---|---|---|---|---|
| `tenant_id` | string | Yes | – | Owning tenant. |
| `sub_tenant_id` | string \| null | No | default sub-tenant | Sub-tenant scope. Required for per-user memory searches. |
| `query` | string | Yes | – | Search terms or natural-language question. Cannot be empty. |
| `source` | `"sources" \| "memories" \| "all"` | No | `"sources"` | What collection to search. `"all"` runs `sources` and `memories` in parallel and merges by `relevancy_score`. |
| `search_by` | `"hybrid" \| "text" \| "vector"` | No | `"hybrid"` | Retrieval method. See [Search methods](#search-methods). |
| `operator` | `"or" \| "and" \| "phrase"` | No | `"or"` | BM25 operator for `search_by: "text"`. Ignored for `hybrid` and `vector`. |
| `mode` | `"fast" \| "thinking"` | No | `"fast"` | Retrieval pipeline. Applies to `hybrid` and `vector` only; ignored for `text`. |
| `max_results` | integer \| null | No | server default | Maximum chunks to return. Start with 10. |
| `alpha` | float `0.0`–`1.0` \| `"auto"` | No | `0.8` | Hybrid weight (`1.0` = pure semantic, `0.0` = pure BM25). Applies to `search_by: "hybrid"` only. |
| `recency_bias` | float `0.0`–`1.0` | No | `0.0` | Boost newer content. |
| `graph_context` | boolean | No | `false` | When `true`, includes the entity/relation graph slice in the response under `graph_context`. |
| `search_forceful_relations` | boolean | No | `true` | Pull author-declared related sources into `additional_context`. **Only takes effect when `mode: "thinking"`** — silently ignored in `fast` mode. |
| `additional_context` | string \| null | No | `null` | Free-form hint to guide retrieval (e.g., "user is on the billing page"). |
| `metadata_filters` | object \| null | No | `null` | Deterministic narrowing before ranking. See [Filters](#filters). |

### Source selection

| Value | Searches | Best for |
|---|---|---|
| `"sources"` *(default)* | Knowledge documents, files, and app sources | Document Q&A, RAG context. |
| `"memories"` | User memories | Personalization and user preferences. |
| `"all"` | Both, merged in one ranked result set | Personalized answers grounded in both shared and user-specific context. |

### Search methods

| Method | Pipeline | Best for |
|---|---|---|
| `"hybrid"` *(default)* | Dense vectors + BM25 keyword | General-purpose retrieval and RAG. |
| `"text"` | BM25 only | Exact terms, compliance lookups, phrase search. Pair with `operator`. |
| `"vector"` | Dense vector only | Pure semantic similarity. |

### Modes

For `search_by: "hybrid"` and `"vector"`:

| Mode | Behavior | When to use |
|---|---|---|
| `"fast"` *(default)* | Single query pass | Real-time chat, autocomplete, simple lookups. |
| `"thinking"` | Multi-query expansion + reranking + forceful-relation context | Complex queries, customer-facing answers, anything where quality matters. |

### Filters

`metadata_filters` runs before ranking. The shape combines two filter scopes:

```json
{
  "metadata_filters": {
    "department": "engineering",
    "region": "us-east",
    "document_metadata": {
      "source": "account_plan",
      "author": "alex"
    }
  }
}
```

| Where | What it matches |
|---|---|
| **Top-level keys** (`department`, `region`) | The source's `tenant_metadata`. Keys must be declared in the tenant's `tenant_metadata_schema` with `enable_match: true`, otherwise they are silently ignored. |
| **Nested under `document_metadata`** | The source's free-form per-document fields. No schema declaration required. |

## Response

```json
{
  "chunks": [
    {
      "chunk_uuid": "policy_v2_chunk_3",
      "source_title": "Compliance Policy v2",
      "source_id": "policy_v2",
      "chunk_content": "Refunds are issued within 30 days...",
      "relevancy_score": 0.91,
      "extra_context_ids": ["pref-tone"],
      "additional_metadata": { "author": "Legal Team" }
    }
  ],
  "sources": [
    { "id": "policy_v2", "title": "Compliance Policy v2", "type": "pdf" }
  ],
  "graph_context": {
    "query_paths": [],
    "chunk_relations": [],
    "chunk_id_to_group_ids": {}
  },
  "additional_context": {
    "pref-tone": {
      "source_title": "User preferences",
      "chunk_content": "Prefers concise answers."
    }
  }
}
```

| Field | Description |
|---|---|
| `chunks[]` | Array of ranked chunks. Preserve server order in your prompt. |
| `chunks[].chunk_uuid` | Stable chunk identifier (string) — use to dedupe across calls. |
| `chunks[].source_id` | The source this chunk belongs to (string). |
| `chunks[].source_title` | Display title of the source (string). |
| `chunks[].chunk_content` | The actual text snippet (string). |
| `chunks[].relevancy_score` | Relevance score (float). Higher is more relevant. |
| `chunks[].extra_context_ids` | Array of keys into `additional_context` for forceful-relation-linked content (`thinking` mode only). |
| `sources[]` | Deduplicated source metadata corresponding to the returned chunks. |
| `graph_context` | Graph slice object, populated when `graph_context: true`. See [GraphContext fields](#graphcontext-fields). |
| `additional_context` | Map keyed by chunk UUID containing related content from forceful relations (`thinking` mode only). |

### `GraphContext` fields

| Field | Description |
|---|---|
| `query_paths[]` | Entity-traversal paths derived from the query. Each path is a list of `{ source, relation, target }` triplets. |
| `chunk_relations[]` | Per-chunk relations grouped by `group_id`. Attach to chunks via `chunk_id_to_group_ids`. |
| `chunk_id_to_group_ids` | Maps each `chunk_uuid` to one or more `group_id`s in `chunk_relations`. |

For formatting `RetrievalResult` into an LLM prompt, see [How to Use API Results](/essentials/v2/api-results).

## Behavior notes

<Info>
**`graph_context` is off by default.** Set it to `true` to receive graph data in the response.
</Info>

<Info>
**`recency_bias` is off by default.** Defaults to `0.0` — no recency boost is applied unless you set it.
</Info>

<Warning>
**`search_forceful_relations` requires `mode: "thinking"`.** In `fast` mode the flag is silently ignored. The server does not error or warn — your `additional_context` will simply be empty.
</Warning>

<Warning>
**`metadata_filters` top-level keys need a schema.** Top-level keys must be declared in `tenant_metadata_schema` with `enable_match: true`. Undeclared keys are silently ignored. To filter free-form per-document fields, nest them under a `document_metadata` key.
</Warning>

<Info>
**Tuning heuristics.**
<ul>
  <li><strong><code>alpha</code></strong>: start at <code>0.8</code>. Lower toward <code>0.3–0.5</code> when the query contains literal tokens (error codes, SKUs, product names). Raise toward <code>0.9</code> for conceptual questions. Use <code>"auto"</code> when query shape varies.</li>
  <li><strong><code>recency_bias</code></strong>: leave at <code>0</code> for static reference material. Set <code>0.2–0.4</code> for mixed content, <code>0.6–0.8</code> for changelogs, news, or status updates.</li>
  <li><strong><code>max_results</code></strong>: start at <code>10</code>. Drop to <code>5</code> for tight context windows; raise to <code>20</code> if you rerank downstream.</li>
</ul>
</Info>
